## 目标

检测 Web 页面的视觉变化，通过截图对比、像素差异检测和布局偏移检测，识别有意的设计变更和无意引入的视觉回归缺陷，确保页面视觉呈现符合预期。

## 操作流程

### 1. 准备阶段：基线获取与环境配置

1. 从 `test_reports` RAG Collection 检索目标页面的历史测试报告，提取：
   - 上次通过的视觉基线截图路径与版本标识
   - 已知的视觉差异白名单（有意变更的已审批记录）
   - 页面视口尺寸历史配置
2. 通过 `playwright_server` MCP Server 配置浏览器环境：
   - 设置标准视口尺寸：桌面端 1920x1080、平板端 768x1024、移动端 375x812
   - 关闭浏览器自动缩放和字体渲染差异（设置 `deviceScaleFactor: 1`）
   - 统一字体渲染（禁用系统字体回退差异）
3. 确定待测页面列表与关键交互状态

### 2. 截图采集

1. 对每个目标页面和视口尺寸组合：
   a. 通过 `playwright_server` 打开页面，等待 `networkidle` 状态（超时 30s）
   b. 等待页面动态内容加载完成（如懒加载图片、异步数据渲染）
   c. 执行全页面截图（`fullPage: true`），保存至测试报告目录
   d. 执行首屏截图（视口范围截图），保存至测试报告目录
2. 对关键交互状态：
   a. 悬停（hover）状态截图：触发元素 hover 后截图
   b. 焦点（focus）状态截图：聚焦输入框后截图
   c. 激活（active）状态截图：按下按钮时截图
   d. 展开/折叠状态截图：展开下拉菜单、手风琴组件后截图
   e. 弹窗/对话框状态截图：打开模态框后截图

### 3. 像素差异检测

1. 将当前截图与基线截图逐像素对比：
   a. 使用结构相似性算法（SSIM）计算整体相似度
   b. 生成像素差异热力图（差异区域高亮标注）
   c. 计算差异像素占比：差异像素数 / 总像素数
2. 对差异区域进行聚类分析：
   a. 将相邻差异像素聚合为差异区块
   b. 过滤微小差异区块（面积 < 阈值，如 10px x 10px，视为抗锯齿/字体渲染差异）
   c. 保留显著差异区块并记录坐标、面积、差异程度

### 4. 布局偏移检测

1. 通过 Cumulative Layout Shift (CLS) 指标检测页面布局稳定性：
   a. 采集页面加载过程中的布局偏移事件
   b. 计算 CLS 分值（Google Core Web Vitals 标准：CLS < 0.1 为良好）
2. 对关键页面元素进行布局位置校验：
   a. 提取基线截图中关键元素的位置坐标（导航栏、CTA 按钮、表单区域）
   b. 对比当前截图中对应元素的位置偏移量
   c. 元素位置偏移超过 5px 标记为布局偏移

### 5. 差异分类与报告

1. 自动分类检测到的视觉差异：
   a. 比对差异白名单，匹配则标记为"有意变更-已审批"
   b. 分析差异区块与页面组件的映射关系，确定影响的组件
   c. 基于差异特征判断分类：
      - 文本内容变化 → 内容变更
      - 颜色/字号变化 → 样式变更
      - 元素位移/大小变化 → 布局变更
      - 元素新增/缺失 → 结构变更
2. 生成差异报告：
   a. 每个差异页面生成对比图：基线截图 | 当前截图 | 差异热力图
   b. 汇总差异统计：总差异区块数、按分类统计、按页面统计
   c. 将差异报告存入 `test_reports` RAG Collection

## 断言策略

### 像素差异阈值

- 整体像素差异占比 < 1%（桌面端视口）
- 整体像素差异占比 < 2%（移动端视口，允许更大的渲染差异）
- 单个差异区块面积不超过视口面积的 5%
- SSIM 相似度 > 0.98（桌面端）/ > 0.95（移动端）

### 布局稳定性

- CLS 分值 < 0.1（Google Core Web Vitals 良好标准）
- 关键元素位置偏移 < 5px
- 关键元素尺寸变化 < 3%
- 导航栏/页脚位置固定不变

### 综合判定

- 所有视口尺寸的像素差异均通过阈值 → 视觉回归测试通过
- 任一视口尺寸存在未白名单的显著差异 → 视觉回归测试失败
- 仅存在已白名单的差异 → 视觉回归测试通过，报告注明有意变更

## 失败处理

### 差异分类处理

- 差异判定为有意变更（设计迭代/内容更新）：
  - 标记为"有意变更-待审批"
  - 生成变更截图对比，等待人工审批
  - 审批通过后加入白名单，更新基线截图
- 差异判定为视觉回归（非预期的视觉变化）：
  - 标记为 `bug` 类别缺陷，优先级根据影响范围判定：
    - 关键元素（导航、CTA、表单）缺失/错位 → `critical`
    - 样式偏差（颜色、字号、间距）但功能不受影响 → `minor`
    - 布局偏移导致内容遮挡或不可交互 → `major`
  - 生成差异报告，包含对比截图和差异详情

### 布局偏移处理

- CLS > 0.1 且 < 0.25：标记为 `bug` 类别缺陷，优先级 `minor`，建议优化
- CLS > 0.25：标记为 `bug` 类别缺陷，优先级 `major`，说明布局不稳定
- 关键元素偏移 > 5px 且 < 20px：标记为 `bug` 类别缺陷，优先级 `minor`
- 关键元素偏移 > 20px 或不可见：标记为 `bug` 类别缺陷，优先级 `major`

### 环境异常处理

- 页面加载超时（>30s）：自动重试 1 次，仍失败则标记为 `environment` 类别缺陷
- 截图采集失败（浏览器崩溃/渲染异常）：自动重启浏览器并重试 1 次，仍失败则标记为 `environment` 类别缺陷
- 基线截图不存在（首次视觉测试）：跳过差异对比，将当前截图保存为基线，标记为"首次基线采集"
- 基线截图与当前视口尺寸不匹配：使用当前视口重新采集基线，记录视口变更

### 知识闭环

- 测试执行完成后，将视觉差异分类结果记录回 `test_reports` RAG Collection，积累视觉回归模式知识
- 审批通过的有意变更自动更新基线截图和白名单，确保后续测试使用最新基线
